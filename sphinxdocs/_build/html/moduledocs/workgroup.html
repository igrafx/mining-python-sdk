Workgroups and Projects¶

Workgroups¶

Workgroups are the first and essential step to accessing your Logpickr data. They are instanciated with a valid workgroup ID and Secret key (which can be found in your Workgroup settings in Process Explorer 360). Workgroups allow you to access the projects created within the workgroup and the associated datasources.

class logpickr_sdk.workgroup.Workgroup(client_id: str, key: str)¶

A Logpickr workgroup, which is used to log in and access projects

property datasources¶

Requests and returns the list of datasources associated with the workgroup

login()¶

Logs in to the Logpickr API with the Workgroup’s credentials and retrieves a token for later requests

project_from_id(pid)¶

Returns a project based on its id, or None if no such project exists

property projects¶

Performs a REST request for projects, then gets project data for any projects that don’t already exist in the Workgroup

Projects¶

A Project object contains the the information about a (drumroll please) Logpickr project! note to self: remove the sarcasm. Each project allows you to get the various graphs it contains, and access its Datasources.

class logpickr_sdk.workgroup.Project(pid: str, owner: logpickr_sdk.workgroup.Workgroup)¶

A Logpickr project

add_file(path)¶

Adds a file to the project

property datasources¶

Requests and returns the list of datasources associated with the project

property graph¶

Performs a REST for the project model graph if it hasn’t already been retrieved

graph_instance_from_key(process_id)¶

Performs a REST request for the graph instance associated with a process key, and returns it

property graph_instances¶

Returns all of the project’s Graph Instances, performing and REST request for any instances that don’t already exist within the project.

property process_keys¶

Queries the datasources to find the different process keys of the project

Datasources¶

Datasources allow you to perform custom SQL requests to the Druid databases containing the data for your projects. Datasources are always obtained through the Workgroups or the Projects, so you shouldn’t need to instanciate one on your lonesome.

class logpickr_sdk.workgroup.Datasource(name: str, dstype: str, host: str, port: str, project: logpickr_sdk.workgroup.Project)¶

An Druid table that can be sent requests by the user

property columns¶

Returns the columns of the datasource

property connection¶

Returns the pydruid connection to the datasource, after initializing it if need be

property cursor¶

Returns the pydruid cursor on the datasource, after initializing it if it doesn’t exist

request(sqlreq)¶

Sends an SQL request to the datasource and returns the results as a pandas Dataframe
